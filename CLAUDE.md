# Visions — Small Business AI Advisor — CLAUDE.md

## Project Summary

**Visions** is an AI-as-a-service web app that helps small business owners discover and adopt
AI tools and agents. Users complete a 3-step intake (business type → pain points → team
size/budget), then receive a personalized AI plan generated by Claude via the Anthropic API.
The app includes task tracking, a booking/consultation flow, a local provider directory, and a
follow-up chat.

**San Diego is the flagship market.** The brand is city-agnostic by design — "San Diego" lives
in config (`src/config/site.js`), not hardcoded throughout the UI — so additional cities or
verticals can be added later without a rewrite.

**Owner:** Troy Latimer / Lati Cooki LLC
**Origin:** Built as a React artifact in Claude.ai (the "SD Small Business AI Advisor"). This
repo is the productionization, rebranded to Visions. The original artifact is archived at
`docs/prototype/sd-biz-ai-advisor.jsx` for reference.

## Current State (scaffold)

This is a **Vite + React + Tailwind** scaffold. The prototype's single 800-line component has
been extracted into a proper component architecture, inline styles converted to Tailwind, and
all SD-specific copy lifted into config. **The backend is not wired up yet** — the Anthropic
call is stubbed behind `src/lib/api.js`, which returns local mock data when `VITE_USE_MOCK` is
`true` (the default). The mock matches the exact JSON shape the real endpoint will return, so
turning on the live API is a config flip plus a deployed Worker, not a component change.

### Design system

- **Palette** (Tailwind `brand-*`, see `tailwind.config.js`): Navy `#0F2B3C`, Sand `#FAF6F0`,
  Ocean `#1A7FB5`, Coral `#E26D5A`, Foam `#6CC4A1`, Slate `#475B6F`, Border `#E2DDD5`,
  Light Blue `#EDF5FA`. Namespaced under `brand-*` to avoid clobbering Tailwind's built-in

- **Typography:** Inter / system-ui stack (`font-sans`).
- **Components:** Cards with 14px radius (`rounded-card`), 1px border. Primary buttons use Ocean
  blue. Effort badges (Quick Start / Some Setup / Deeper Build). Progress dots for intake steps.
- **Vibe:** Coastal-professional. Warm but not corporate. San Diego energy without being kitschy.

## Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| API model | claude-sonnet-4-6 | Fast enough for interactive use, good at structured JSON |
| Response format | JSON-only prompt (no markdown, no backticks) | Reliable parsing |
| max_tokens | 4096 (plan), 2048 (chat) | Plan JSON needs ~2000-2500 tokens; lower caused truncation |
| State management | React useState in `App.jsx` | Single-page flow; sufficient until routing is added |
| Persistence | localStorage shim (`lib/storage.js`) | Drop-in surface; replace with D1 via the Worker |
| Styling | Tailwind CSS | Migrated off the prototype's inline styles |
| Config-driven location | `src/config/site.js` | Keeps "San Diego" out of components for future markets |

## Prompt Engineering Notes

When the backend Worker is built, the plan generation prompt should be tuned for:

- Structured JSON output (no markdown wrapping). Shape (see `src/lib/mockData.js` for the exact
  contract): `headline`, `quick_wins[]` (title, description, tools[], effort, monthly_cost, task),
  `agent_opportunity` (title, description, impact), `next_step`.
- Local awareness from `SITE.location.flavor` (tourism, border trade, military, beach culture).
- Budget-aware, real tool names (ChatGPT, Claude, Zapier, HubSpot, Square, Toast, Calendly, Canva…).
- Each quick win includes a concrete `task` field for the task board integration.
- The `effort` field maps to UI badges: easy → "Quick Start", medium → "Some Setup",
  advanced → "Deeper Build".

The follow-up chat prompt carries the full profile (business type, team size, budget, pain
points) plus the plan headline, so responses stay contextual without regenerating the plan.
