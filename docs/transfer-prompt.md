## What Visions is
An AI-as-a-service app: small-business owners do a 3-step intake (business type → pain points →
team size/budget) and get a personalized AI plan generated by Claude. It has task tracking, a
booking flow, a local provider directory, follow-up chat, and shareable `/plan/:id` links. San
Diego is the flagship market (kept in config, not hardcoded).

## Stack & layout
- **Frontend:** Vite + React + Tailwind in `src/` (routes `/` and `/plan/:id`). Calls the API
  via `src/lib/api.js`; production build (`.env.production`) hits the real API, `npm run dev`
  uses mock data.
- **Backend:** zero-dependency Cloudflare Worker in `worker/src/` (raw `fetch`, no SDK). Serves
  `/api/*` and the static frontend (Static Assets, `run_worker_first=["/api/*"]`, SPA fallback).
- **Data:** Cloudflare D1 (`schema.sql`). `node --test` covers the Worker logic (`npm test`).

## Key facts
- **Model:** `claude-sonnet-4-6` (env `ANTHROPIC_MODEL`) — the project's deliberate choice.
- **Generation path:** `wrangler.toml` sets `USE_MANAGED_AGENT="true"`, so the Worker drives a
  Managed Agent — `agent_011CZtoh5iVJGPjFmdXkSDzo` in environment `env_015fkJc7jYAiMT6vN1DMPMBt`
  — per request (create session → send message → poll to idle → parse). Set it to `"false"` to
  use the Messages API with structured outputs instead (faster, schema-guaranteed JSON). **Both
  paths authenticate with the `ANTHROPIC_API_KEY` Worker secret**, and that key must belong to
  the Anthropic workspace that owns the agent + environment.
- **Key handling:** `ANTHROPIC_API_KEY` is a Worker secret, never in the client bundle.

## How to run/build/test locally
```bash
npm install
npm run dev          # frontend on mock data (no key) — http://localhost:5173
npm test             # Worker tests (node --test)
cp .dev.vars.example .dev.vars   # add ANTHROPIC_API_KEY for full-stack local dev
npm run db:schema:local && npm run build && npm run worker:dev   # http://localhost:8787
```

## Constraints
- Worker stays **zero-dependency, no framework** (ESM module-worker format; raw `fetch`).
- Tests are **`node --test`**. The Anthropic key is **server-side only**.

## Gotchas
- The Managed-Agent path loses the Messages API's schema-guaranteed JSON (plan JSON is
  prompt-enforced + parsed with fence tolerance) and is heavier (a session per request). If
  reliability/latency matter more than using the agent, flip `USE_MANAGED_AGENT="false"`.
- Landing stat numbers and provider names are aspirational placeholders (see CLAUDE.md).
- Branch of record: `main` (mirrors `claude/blissful-wright-cb3hpp`). Don't create a PR unless asked.
